Programación Estructurada

Introdución.

La Programación Orientada a Objetos supone un cambio en la concepción del mundo de desarrollo de software, introduciendo un mayor nivel de abstracción que permite mejorar las características del código ﬁnal. De manera muy básica, las aportaciones de este paradigma se pueden resumir en:

  • Conceptos de clase y objeto, que proporcionan una abstracción del mundo centrada en los seres, objetos o cosas y no en los verbos.
  • Los datos aparecen encapsulados dentro del concepto de clase. El acceso a los datos se produce de manera controlada e independiente de la representación ﬁnal de los mismos. Como consecuencia, se facilita el mantenimiento y la evolución de los sistemas, al desaparecer las dependencias entre distintas partes del sistema./li>
  • Mediante conceptos como la composición, herencia y polimorﬁsmo se consigue simpliﬁcar el desarrollo de sistemas. La composición y la herencia nos permiten construir clases a partir de otras clases, aumentando en gran medida la reutilización.

Importancia del paradigma orientado a objectos

El paradigma orientado a objectos hace más fáciles las cosas al abstraer lo que se encuentra en la vida real y manipularlo como lo haría naturalmente una persona. Los objetos que observamos cotidianamente tienen propiedades con las cuales los identificamos: tienen un color, dimensiones, texturas (lisas, ásperas, etc.), estados (frío, caliente, encendido, apagado, etc.), entre muchas otras. Y también cumplen con una funcionalidad: en el caso de un carro se le incorpora un sistema de arranque, es posible mover el carro con el acelerador, utilizar el limpiaparabrisas, etc.

Con este paradigma es posible utilizar esta forma de ver el mundo y aplicarlo a la programación. Se pueden crear ‘objetos’, determinar cuales son sus propiedades utilizando los tipos de datos que ya conocemos (String, int, double, boolean, etc.), las cuales a futuro nos darán información sobre el objeto, también es posible modificar esas propiedades mediante funciones (métodos) que están asociadas a él, entre otras características.

Si pensamos en una de las aplicaciones, se encuentra la facilidad de representar un personaje de un videojuego. Esta instancia al momento de su creación tendrá valores por defectos que se asocian con el rol y nivel que tendrá el personaje en ese momento, lo que serían sus propiedades: vida, poder asociado, experiencia, etc. También será posible manipular el personaje mediante métodos asociados a él. Podrá correr, atacar, saltar, subir de nivel, recibir daños, obtener vida. Cada método modificará una propiedad específica, ya sea la vida, poder asociado, experiencia ganada, etc. De igual forma es posible crear diferentes partidas, por cada usuario que tiene acceso al juego, por lo que se necesitan crear más objetos que tendrán la misma clase en común, ‘Personaje’.

De esta manera es más fácil agrupar el código por archivos que representen y cumplan con un solo propósito. Dentro de las ventajas de este paradigma se encuentran:

  1. Los componentes se pueden reutilizar.
  2. Facilidad de mantenimiento y modificación de los objetos existentes.
  3. Una estructura modular clara se puede obtener, la cual no revelará el mecanismo detrás del diseño.
  4. Se proporciona un buen marco que facilita la creación de rica interfaz gráfica de usuario aplicaciones (GUI).
  5. Se acopla bien a la utilización de bases de datos, debido a la correspondencia entre las estructuras.

Atributos y métodos

Las variables como comúnmente las conocemos se tornan en atributos cuando éstos se encuentran en una clase. De la misma manera, las funciones se llaman métodos de dicha clase. Los atributos y funciones se declaran de la misma manera que se haría en otro lugar: 

                
                    public class Persona {
                        String nombres;
                        String apellidos;
                        int edad;  
                        double estatura;
                       double peso;
                     }

Dichos atributos describen a cualquier persona. No importa el rol que juegue esa persona, si es un menor de edad, adulto, estudiante o profesional, todos cumplen con las mismas características por lo tanto es posible instanciar dicha clase y crear objetos que tendrán las mismas características.

Ahora con los métodos de la clase Persona: 

                
                    public class Persona {
                        String nombres;
                        String apellidos;
                        int edad;  
                        double estatura;
                        double peso;
                        
                        public boolean esMayorDeEdad(){
                          if(this.edad >= 18){
                           return true;
                          }
                          else{
                          return false;
                         }
                        }	    
                      }

Existe una función llamada constructor de la clase, la cual, al momento de instanciar un objeto, se le pasan por argumento los valores deseados para inicializarlo. En la clase se vería así: 

                
                    public class Persona {
                        String nombres;
                        String apellidos;
                        int edad;  
                        double estatura;
                        double peso;
                      
                        public Persona(String nombres, String apellidos, int edad){
                          this.nombres = nombres;
                          this.apellidos = apellidos;
                          this.edad = edad;
                        }
                        
                        public boolean esMayorDeEdad(){
                          if(this.edad >= 18){
                           return true;
                          }
                          else{
                          return false;
                         }
                        }	    
                      }

El constructor sigue la siguiente sintaxis: 

                
                    public ( nombre_parametro1,  nombre_parametro2){
                        //se colocan los valores que tendrá el objeto al ser instanciado
                        }

Desde la función main dónde hacemos la instancia:

                
                    Persona persona1 = new Persona("Rodrigo", "Corvera Diaz", 20);

Para poder utilizar los métodos de dicho objeto se emplea la notación del punto:

                
                    if(persona1.esMayorDeEdad())
                    {
                    System.out.println("Nos vamos pal tunco");
                    }
                    else
                    {
                    System.out.println("No, gracias, no tengo edad para salir");
                    }

De la forma en la que hemos escrito actualmente la clase, es posible acceder a los atributos directamente por medio de la notación del punto: por ejemplo: 

                
                    if(persona1.edad >= 18 )
                    {
                    System.out.println("Nos vamos pal tunco");
                    }
                    else
                    {
                    System.out.println("No, gracias, no tengo edad para salir");
                    }

Clases y Objetos

El mundo está lleno de objetos: el automóvil, un edificio, una lavadora, un empleado, etc. El paradigma de programación a objetos proporciona las abstracciones necesarias para resolver una problemática a partir de la implementación de un sistema de software que conlleva a una mejor percepción del mundo real.

Mediante la programación orientada a objetos podemos desglosar un problema en subgrupos. Estos subgrupos pueden traducirse en unidades autocontenidas llamadas objetos, los cuales responden a determinadas características y operaciones. Antes de la creación de un objeto, se debe de definir en primer lugar su estructura general (Plantilla) la cual recibe el nombre de clase. Dentro de una clase definimos las características generales que posee un objeto, esto se logra mediante la abstracción de datos del problema considerado.

Arquitectura de clase en java 

                
                    public class Estudiante {
    
                        /*
                            Atributos o propiedades de la clase.
                            Descripción: Propiedades que caracterizan el objeto que se esta analizando.
                        */
                        
                        private String carnet;
                        private String correo;
                        private String nombre;
                        private String apellido; 
                        private int edad;
                        
                        private Calendar fechaDeIngreso;
                        
                        
                        /*
                            Método constructor.
                            Descripción: Se encarga de inicializar un objeto tipo Estudiante.
                        */
                        
                        public Estudiante (String carnet, String correo, String nombre, String apellido, int edad, String fechaDeIngreso) {
                                this.edad = edad;
                                this.carnet = carnet;
                                this.correo = correo;
                                this.nombre = nombre;
                                this.apellido = apellido;
                                
                                SimpleDateFormat dateFormat = new SimpleDateFormat("dd-MM-yyyy");
                                
                                Date ingreso = dateFormat.parse(fechaDeIngreso);
                                this.fechaDeIngreso = Calendar.getInstance();
                                this.fechaDeIngreso.setTime(ingreso);
                        }
                    
                        /* Setters & Getters */
                        
                        public String getCarnet() {
                            return carnet;
                        }
                    
                        public void setCarnet(String carnet) {
                            this.carnet = carnet;
                        }
                    
                        public String getCorreo() {
                            return correo;
                        }
                    
                        public void setCorreo(String correo) {
                            this.correo = correo;
                        }
                    
                        public String getNombre() {
                            return nombre;
                        }
                    
                        public void setNombre(String nombre) {
                            this.nombre = nombre;
                        }
                    
                        public String getApellido() {
                            return apellido;
                        }
                    
                        public void setApellido(String apellido) {
                            this.apellido = apellido;
                        }
                    
                        public int getEdad() {
                            return edad;
                        }
                    
                        public void setEdad(int edad) {
                            this.edad = edad;
                        }
                    
                        public Calendar getFechaDeIngreso() {
                            return fechaDeIngreso;
                        }
                    
                        public void setFechaDeIngreso(Calendar fechaDeIngreso) {
                            this.fechaDeIngreso = fechaDeIngreso;
                        }
                        
                        /* Métodos de la clase (Acciones) */
                        
                        public void presentarme () {
                            JOptionPane.showMessageDialog(null, "Hola, mi nombre es: " + this.getNombre() + this.getApellido(), "Saludo", JOptionPane.INFORMATION_MESSAGE);
                        }
                        
                        public void saludar (String nombre) {
                            JOptionPane.showMessageDialog(null, "Hola " + nombre + ", ¿Cómo te va?", "Saludo", JOptionPane.INFORMATION_MESSAGE);
                        }
                    }
